# Scrabble Vision

A fast, offline Scrabble board scanner. Takes a photo of a board and outputs a 15x15 letter grid.

Built as a standalone computer vision + ML pipeline with no external API calls for inference.

## Pipeline

1. **Corner Detection** — User selects the 4 board corners (interactive click or auto-detect), then perspective-warps to a perfect square
2. **Grid Segmentation** — Equal 15x15 division (correct because physical boards have uniform cell spacing)
3. **Tile Presence Detection** — Determines which cells have wooden tiles vs exposed board squares, using color/texture heuristics + hardcoded bonus square positions
4. **Letter Classification** — Lightweight CNN classifies only the tile cells (A-Z, blank, empty)
5. **Editable Output** — Returns a 15x15 matrix with confidence scores; uncertain cells are flagged for user review

## Quickstart

```bash
uv init && uv add opencv-python numpy torch torchvision Pillow onnx onnxruntime

# Generate synthetic training data
uv run generate_tiles.py

# Train the classifier
uv run train.py

# Scan a board (interactive corner selection)
uv run scan.py --image data/raw_boards/board.jpg --interactive --debug

# Reuse saved corners
uv run scan.py --image data/raw_boards/board.jpg --corners corners.npy --debug

# Also extract detected words
uv run scan.py --image data/raw_boards/board.jpg --interactive --debug --words
```

## Design Decisions

- **Corner selection → perspective warp → equal division** — validated by multiple reference projects; grid line detection is unnecessary because the board has uniform spacing
- **Hardcoded bonus square positions** — bonus squares are part of the game rules, not the board design, so they're identical on every standard board; this makes tile detection robust across board styles
- **Tile detection before classification** — empty board squares have printed bonus text that confuses the letter classifier; filtering them out first eliminates the biggest source of false positives
- **Synthetic training data** — renders tiles with augmentation (rotation, noise, shadows) to bootstrap the classifier without photographing thousands of tiles
- **Lightweight CNN (~360K params)** — targets <50ms full-board inference on CPU
- **ONNX export** — enables future Rust/mobile deployment via `ort`

## Web App

A mobile-friendly web interface for testing the scanner on a phone.

```bash
uv add fastapi uvicorn python-multipart
uv run uvicorn server:app --host 0.0.0.0 --port 8000
```

Open `http://<your-local-ip>:8000` on your phone (same WiFi). Camera feed with grid guide → capture → drag corners to align → scan → editable board result.
